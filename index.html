Introduction

HTML(Hyper Text Markup Language) is the standard markup language for creating web pages. It describes the structure of Web pages using a markup language comprised of elements called tags. These tags are the building blocks of HTML pages by labelling pieces of content such as "heading", "paragraph", "table", and so on Browsers do not display the HTML tags, but use them to render the content of the page control over them.

HTML is closely associated with many other languages as when combined, the scope of design opportunities available increase exponentially. CSS(Cascading Style Sheet) and Javascript are the most common, with CSS used to alter the appearance of a page and Javascipt used to add additionaly functionality to elements.

HTML can be created/ modified using a variety of editors; included within in these are:

  • Notepad/ Wordpad
  • Codepen (the editor this web page was created in)
  • JSFiddle
  • as well as many others
A Basic Example

To get started with writing HTML, open a Notepad file and write your first "Hello world" webpage:

<!DOCTYPE html> 
<html>
  <head>
    <title> My First Page </title>
  </head>
  <body>
    <h1>Hellow World Heading</h1>
    <p>Hello World</p>
  </body>
</html>

Select the code above, paste it into the file, save it as a .html file and then open it with your favourite web browser. See the results unfold!

Things To Know

All HTML tags start with <, end with > and contain the tagname of the element between. Most HTML Elements are comprised of two tags: an opening tag and a closing tag. The opening tag contains additional info if required like the id of the tag, what class the tag belongs to, styling information or other details pertinent to the element. The closing tag must have forward slash before the tagname.

The first line in the example, <!DOCTYPE html> details the document as being an HTML5 web page. The <html> element defines the root element of the entire HTML Document. The <body> element contains the visible page content. The <h1> element defines a large heading. The <p> element defines a paragraph element.

All HTML elements have attributes. THey are set in the opening tag of an element after the tagname. Available attributes vary between elements, with some requiring them to achieve the intended effect of the element. Elements like anchor links, images and different user inputs are great examples. There also also many attributes that every element has, including some important ones:

  • id, used for identifying a single element
  • class, used for grouping elements which share styling information
  • style, used for styling an element(CSS)
Head VS Body

You may have noticed in the example in the "A Basic Example" section that there is both a <head> and a <body> elemenr, Each element has a very important purpuse within a web page. The head element has a variety of uses. These uses include:

  • giving a title to a web page (the name displayed in the browser tab the page is open in)
  • proving keywords to search engines to categorise the page(kif the page is online)
  • linking external CSS files to be used in the page
  • linking external Javascript files to be used in the page

as well as many other important uses.

The body element contains all of the content that is displayed in the browser(with the exception of the title). All web pages must contain a body element if the web page has any visual content.

Headings & Paragraphs

Headings are the focal elements to start a page or section with. The general opening tag of a heading is <h[number]>,with number specifying the importance of the heading. A h1 heading is the most important on the page and should be reserved for the title of the website or the title of the page. You increase the number as you use headings in sections, sub-sections etc. You should never skip numbers in sequence, i.e from 1 to 6 without 3, and you can have multiple equivalent headings for equally important section (two h2's).

Paragraphs are the basic way to display text in your browser. By adding a paragraph element with a <p> tag, you can display the text you specify between the opening and closing tags on your page.

<p> Here is my first paragraph </p>

Note that with normal paragraph elements any odd spacing features or line breaks you may add to your text are ignored. If you want to keep them in, you need to use the preformatted text tag <pre>, which will preserve the way you typed out your text in the code.

Format Your Text

You may have seen throughout this page how many important words have been italisced. This an example of text formatting. There are various different elements within HTML to format your text. The use of such elements can help with both styling as well as highlighting important parts of the text. These elements include:

  • <b>, used for making text bold
  • <i>, used for making text italic
  • <strong>, used for making text bold but with added semantic importance
  • <em>, used for making text italic but with added semantic importance
  • <super>, used for making text superscript
  • <sub>, used for making text subscript

Note: You may have noticed how both <b> & <strong> and <i> & <em> appear to be the same. They both achieve the same styling effect but the use of <strong> & <em> over their counterparts is suggested because the use of these elements changes how text is recognised by both screen readers(for blind people) or small devices.

Lists

There are two types of lists within HTML: unordered lists and ordered lists. Unordered lists <ul> </ul> are for lists where the order of the elements in the list have no meaning (semantically similar to a set) while ordered lists <ol> </ol> are for when the order of elements has meaning(rankings etc). Both lists share the same tags for list elements <li> </li>.

Unordered List example:

<ul>
      <li> item1 </li>
      <li> item2 </li>
      <li> item3 </li>
</ul>

Ordered List example:

<ol>
      <li> item1 </li>
      <li> item2 </li>
      <li> item3 </li>
</ol>
Comments

Comments are a helpful to to explain you code to those who may read it. Your comments will not appear on the actual web page but only when the actual HTML code is read. You can add a comment to a line by typing <!-- where the comment starts and end the comment with -->.

<p> Here is some interesting text <!-- Here is a comment --> </p>
General Containers

When your web page grows bigger, the ability to divide and group content meaningfully within the HTML code becomes more important. HTML offers many elements which are for this explicit purpose, with some intented to represent a certain type of geberal page content.

The most general container is the <div> element. The div element is often used as a container for other HTML elements to style them with CSS or to perform certain tasks with JavaScript.

HTML5, the version which this page is teaching, introduced new elements to represent certain general web page content. These help not only in grouping content on the page, but also help semantically to detail which sections of code relate to certain page elements. These elements include:

  • header, represented by <header>, used for containing introductory page information (navigational/logo/business name etc) or page section info.
  • nav, represented by <nav>, represents a set of navigation links. It is is intended only for major block of navigation links, like navigation bars.
  • main, represented by <main>, used for containing all page info not repeated on multiple pages of a site. This includes sidebars, navigation links, copyright information, site logos, and search forms.
  • article, represented by <article>, represents individual, self contained content. Frequently used for blog/forum posts, website comments, news stories etc.
  • section, represented by <section>, is used for thematic grouping of content. Used to represent chapters, tabbed pages in a tabbed dialog box, or numbered sections of a thesis etc.
  • footer, represented by <footer>, used for containing information about a section or page(typically things like top links, author info, copyright info etc). Usually at the bottom of a page/section.

Link Your Page

Adding links to your web page through the HTML anchor tag is a crucial step. You can link to pages outside your web site as well as other parts of your site. This even includes jumping to different places on the same page! Here is an example of an anchor tag in action:

<a href="www.google.com"> Google Homepage </a>

This anchor tag links the text "Google Homepage" to the actual Google home page. This is done by specifying the href attribute of the tag. To link to a different place on the same page, you must specify the href attribute of the anchor tag to be in the form of # id of element to link to. An example of this is shown below, with the text linking to the references section of a page where the first element of said section has the id of references. 

...
<a href="#References"> References </a>
...
<p id="References"> Here are all my references </p>
...
Images

A picture is worth a thousand words.

Images are a vital part of most web pages. They are represented in HTML by the img element. You need to set the src attribute of the element the URL of the image(if you are preparing a page online this will be the address where the image is stored, otherwise if you are testing a page locally it will be the file system link of it). An example of adding an image to a page through a img tag is as follows:

<img> src="C:\Users\Public\Desktop\image.png" </img>
